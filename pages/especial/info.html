Entrevista al creador de la APP

Preguntas y respuestas

Preguntas y respuestas sobre la aplicación de la expo 2018 ¿Que es la aplicación?

La aplicación es una PWA (progressive web app), está basada en tecnologías web HTML5

¿Cuál es el objetivo?

Brindar información breve de los sectores y actividades que se desarrollan en la expo

¿Quien la desarrollo

Yo, Tomas Cichero de 5to año 2018. Y nadie más participo en el desarrollo. Tuve la colaboración de Catalina Lanfranchi para obtener información de algunos sectores de la expo.

¿Cuanto tiempo tardo/aron?

Empecé a desarrollar la aplicación a principio de año, de apoco la fui desarrollando durante el año hasta hoy.

¿Cuanto costo?

El costo monetario fue nulo ya que se utilizaron tecnologías open source para el desarrollo de la app.

¿Cuanto ayudo el colegio?

El profesor de informática Diego Montoto me ayudó con la obtención de la información necesaria sobre la expo para mostrar en la aplicación. Y el profesor Milton Torasso me ayudo con cuestiones técnicas del alojamiento de la aplicación en el servidor del colegio.

¿Que se puede hacer en la aplicación?

La aplicación tiene una lista de todos los sectores que van a haber en el expo, las actividades y horarios que sucederán en el expo y un mapa interactivo con la posición en el colegio de cada uno de los sectores y actividades.

¿Como la diseñaron?

La aplicación la diseñé basándome en un framework de material design para poder tener una interfaz sencilla y amigable para el usuario.

¿Es la única aplicación que hicieron?

En la expo 2015 habíamos creado otra aplicación aunque mucho menos avanzada que esta. Esta aplicación era una app nativa de Android, creada con MIT AppInventor.

¿Como la descargo?

La forma de descargarla es entrando al link de la aplicación www.henryford.esc.edu.ar/expo o al link bit.ly/hfexpo2018. Se puede usar directamente desde el navegador, o se puede “instalar” como una aplicación independiente usando la opción del navegador llamada “agregar a la pantalla principal”, esto te agrega un icono que te abre la aplicación en pantalla completa.

¿Funciona en todos los teléfonos?

Si, cualquier teléfono con un navegador web moderno

¿Como contacto al que la creo?

En la app, en el menú, la parte de “Créditos”

¿Como está programada?

La aplicación está programada con un editor de código llamado “Atom”, que además tiene integración con GitHub.

¿Que lenguaje usa?

Utiliza HTML, CSS y JavaScript. No requiere backend especifico.

¿La información es actualizable?

¿Necesita de un servidor?

Si, al ser una aplicación web, se requiere acceder desde un navegador a la pagina web de la app, pero después de la primera carga se puede volver a cargar la aplicación cuantas veces se quiera sin necesidad de utilizar internet.

¿Que servidor usan?

La aplicación está alojada en el servidor de la escuela.

¿Necesita wifi?

Se necesita alguna conexión a internet para la primera carga, pero se puede utilizar sin conexión a internet (como cualquier app). La conexión a internet no tiene que ser wifi necesariamente.

¿Porque si/no necesita wifi?

La aplicación pesa muy poco y con conexión de datos se puede cargar perfectamente, aunque se tenga una muy lenta conexión a internet. No se requiere internet para poder usarla, pero si para cargarla por primera vez.

¿Como hago para editarla?

Se puede contribuir en el proyecto en GitHub (https://github.com/Martoxdlol/HF-Expo-2018-APP).

¿Seria normal que se caiga?

No porque la aplicación no tiene backend especifico, esto quiere decir que simplemente el servidor envía los archivos necesarios cuando se carga la página, solo se podría caer si hay algún fallo en el servidor, pero esto no depende de la aplicación.

¿Como lo arreglo en ese caso?

Restableciendo el servidor.

¿Cuanto tiempo invertiste?

No sé exactamente, pero muchas horas desde que empecé a desarrollarla

¿Cuanta plata invertiste?

No tuve que invertir plata ya que utilicé solo tecnologías gratuitas y/o open source.

¿Sobre que sistemas funciona?

Funciona sobre cualquier navegador moderno, por ejemplo: Google Chrome, Safari, Microsoft Edge, internet Explorer, Opera, etc. Se puede utilizar cualquier sistema operativo con alguno de estos navegadores.

¿Breve resumen de como la programaste?

Utilicé un framework CSS y Javascript llamado materializecss para el diseño de la interfaz. El desarrollo del código lo hice utilizando un editor de código llamado “Atom”. La programé creando una base sobre la cual luego se cargan todos los archivos con la información del a expo. Esta base permite cargar archivos y mostrarlos con links dentro de la app que se marcan con “#!” y luego el nombre de la pagina dentro de la app. Luego de tener la base para visualizar archivos cree una base para mostrar ordenadas y correctamente las actividades que se realizaran en la expo. Para el mapa utilice una api llamada Map Box la cual es gratuita. Luego de tener la base pasé a la segunda etapa de cargar la información. La aplicación no tiene ningún sistema especial para cargar información, simplemente se cargan los archivos HTML de cada sector y se configuran actualizando una lista donde figuran todos los sectores y archivos.
